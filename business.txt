Below is a **0 → 100% OpenCode execution plan**.

Use it exactly like this:

1. Create docs and rules.
2. Audit the current whiteboard app.
3. Build one slice at a time.
4. Run checks after every slice.
5. Commit after every slice.
6. Do not let OpenCode expand scope randomly.

---

# Ground rules for using OpenCode

Do **not** give OpenCode one giant “build everything” prompt.

Use this pattern:

```text
Plan → Implement slice → Test → Fix → Commit → Next slice
```

Your current app already has the whiteboard working, so the job is now to build the **private planning suite around it**.

# Phase 1 — Create permanent project rules

Give OpenCode this prompt:

```text
Create a project instruction file called AGENTS.md.

This app is called SiteSledge Command Center.

It is a private internal business planning app for one owner.

Business context:
- I run a GoHighLevel-based local business marketing system.
- I sell a low-ticket recurring marketing foundation system to local service businesses.
- Monthly price target: $97 to $297.
- Core pillars: SEO website, automated Google review requests, missed-call text-back automation, and database reactivation.
- Fulfillment is standardized and later delegated to VAs.
- Target: 100+ low-churn clients.

Product rules:
- This is NOT a SaaS product.
- Do NOT add signup.
- Do NOT add registration.
- Do NOT add client accounts.
- Do NOT add public users.
- Do NOT add multi-tenant logic.
- This is for my private use only.

Access rules:
- Use a single private password gate or single admin access.
- Prefer a password gate using an environment variable.
- Password must come from environment variable.
- Protect all planning pages.
- No registration page.
- No public account creation.

Tech stack:
- Keep the existing stack.
- Keep the existing working Excalidraw whiteboard.
- Use Next.js App Router.
- Use TypeScript.
- Use Tailwind.
- Use PostgreSQL and Prisma if already installed.
- Use Zod for validation.
- Use Docker Compose if already installed.
- Use pnpm.

Engineering rules:
- Do not break the existing whiteboard.
- Do not rewrite working systems unless necessary.
- Do not use tldraw.
- Do not use paid-license canvas SDKs.
- Do not use localStorage as the main database.
- Do not fake persistence.
- All business records must persist in PostgreSQL.
- Use TypeScript everywhere.
- Avoid `any` unless absolutely necessary.
- Use Zod validation for inputs.
- Keep components reusable.
- Keep server/database logic separate from UI components.
- Do not leave TODOs for core functionality.
- Do not remove features to make the build pass.

Build process:
Before each implementation:
1. Read AGENTS.md.
2. Read docs in /docs.
3. Confirm the slice scope.
4. Implement only that slice.
5. Run checks.
6. Fix errors.
7. Update docs if implementation differs.

Required checks after each slice:
- pnpm lint
- pnpm typecheck if available
- pnpm build

If typecheck script does not exist, add a proper typecheck script.

When finished with any slice, report:
- What changed
- Files changed
- Commands run
- Errors found
- Errors fixed
- Remaining limitations
```

Then commit:

```bash
git add .
git commit -m "Add project agent instructions"
```
